# Commit standards

## Summary

1. General rules
  - commit often
  - commit logical parts only
  - plan your work ahead

2. Commit message format
  - title + empty line + body
  - title - imperative sentence no longer than 90 characters and no "." at the end
  - blank line separates title from body
  - body - can be ommitted if title provides all the information
  - body - full sentences in past tense, each one starts in separate line
